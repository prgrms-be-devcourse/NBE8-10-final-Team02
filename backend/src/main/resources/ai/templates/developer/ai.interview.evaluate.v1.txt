평가 축:
- relevance
- structure
- evidence
- domainDepth
- clarity

질문 유형별 보정:
- experience: 문제-행동-결과 연결성 강조
- technical_cs: 개념 정확성과 핵심어 강조
- technical_stack: 선택 이유와 trade-off 강조
- behavioral: 상황 설명과 협업 맥락 강조

점수 기준 (totalScore 및 개별 score 모두 적용):
- 0-20: 질문과 무관하거나 핵심 개념이 완전히 누락된 답변
- 21-40: 관련은 있으나 구체성이 부족하고 표면적인 답변
- 41-60: 기본 요소를 충족하나 깊이 또는 구조가 부족한 답변
- 61-80: 구체적이고 논리적이며, 일부 보완이 필요한 답변
- 81-100: 탁월한 구조와 근거를 갖추고 실전 활용 가능한 수준의 답변

규칙:
- isSkipped=true면 score는 0으로 부여한다.
- 답변이 질문과 완전히 무관하거나, 의미 없는 문자열(반복 문자, 무작위 입력 등)이면 score는 0~5로 부여한다.
- 질문의 주제를 언급했으나 실질적 내용이 없으면 score는 6~20으로 부여한다.
- evaluationRationale은 1~2문장으로 짧게 쓴다.
- 태그는 tagMaster 안에서만 선택한다.
- 태그 수는 최대 3개다.
- summaryFeedback은 전체 답변 경향을 요약해야 한다.

응답 JSON 구조 (반드시 이 키 이름과 구조를 정확히 따라라):
{
  "totalScore": 75,
  "summaryFeedback": "전체 답변에 대한 종합 피드백",
  "answers": [
    {
      "questionOrder": 1,
      "score": 80,
      "evaluationRationale": "해당 답변에 대한 평가 근거",
      "tagNames": ["태그1", "태그2"]
    }
  ],
  "qualityFlags": []
}
