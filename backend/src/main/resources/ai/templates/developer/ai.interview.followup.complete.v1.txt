목표:
- answeredThreads 전체를 보고 정말 필요한 경우에만 completion-stage follow-up을 생성한다.
- follow-up은 0개 또는 1개를 기본값으로 하고, 정말 필요한 경우에만 2개까지 허용한다.
- 애매하면 생성하지 않고 빈 배열 []을 반환한다.

판단 규칙:
- 각 thread의 rootQuestion, rootAnswer, runtimeFollowupQuestion, runtimeFollowupAnswer를 함께 읽는다.
- rootAnswer와 runtimeFollowupAnswer까지 봐도 핵심 이유, 결과, 판단 기준 중 하나가 여전히 부족할 때만 생성한다.
- runtimeFollowupQuestion/runtimeFollowupAnswer가 있으면 rootAnswer와 함께 하나의 thread로 이해한다.
- 이미 물은 포인트를 거의 반복하게 되면 생성하지 않는다.
- 전혀 새로운 기술 주제로 점프하지 않는다.
- parentQuestionOrder는 반드시 answeredThreads[].tailQuestionOrder 중 하나만 사용한다.
- 같은 parentQuestionOrder를 두 번 이상 사용하지 않는다.
- 출력 배열은 parentQuestionOrder 오름차순으로 정렬한다.

출력 규칙:
- 반드시 JSON object 하나만 출력한다.
- 줄바꿈과 들여쓰기는 허용하지만, 유효한 JSON object만 출력한다.
- 생성하지 않는 경우에도 반드시 `followUpQuestions` 키를 유지하고 빈 배열 []을 반환한다.
- `followUpQuestions`는 0개, 1개, 또는 정말 필요한 경우의 2개만 허용한다.
- 각 item의 `questionType`은 항상 `follow_up`이다.
- 각 item의 `difficultyLevel`은 항상 `medium`이다.
- `questionText`는 한 문장만 작성한다.
- `questionText`는 최대 60자 이내로 유지한다.
- 배경 설명, 완곡한 서론, 부연 설명 없이 바로 질문만 쓴다.
- `qualityFlags`는 [] 또는 ["low_context"]만 사용한다.
- 지정된 키 이름 외 다른 키를 추가하지 않는다.

금지:
- 입력에 없는 order를 parentQuestionOrder로 선택
- 같은 thread에 completion-stage follow-up을 2개 이상 생성
- 예/아니오형 단답 질문
- 기존 질문을 거의 그대로 반복
- 전혀 새로운 기술 주제로 점프
- 마크다운, 코드 블록, 설명 문장, 서론, 후기 출력

아래 JSON 예시처럼 여러 줄 JSON으로 출력해도 된다.
단, JSON 바깥의 텍스트를 추가하면 안 된다.

응답 JSON 구조:
{
  "followUpQuestions": [
    {
      "questionType": "follow_up",
      "difficultyLevel": "medium",
      "questionText": "그 판단 기준을 예시와 함께 설명해주실 수 있나요?",
      "parentQuestionOrder": 3
    }
  ],
  "qualityFlags": []
}

생성하지 않는 경우:
{
  "followUpQuestions": [],
  "qualityFlags": ["low_context"]
}

생성 예시:
입력:
{
  "jobRole": "Backend Developer",
  "companyName": "Example Corp",
  "answeredThreads": [
    {
      "tailQuestionOrder": 2,
      "rootQuestion": {
        "questionOrder": 1,
        "questionType": "project",
        "questionText": "프로젝트에서 어떤 기준으로 접근 방식을 선택하셨나요?",
        "difficultyLevel": "medium"
      },
      "rootAnswer": {
        "answerText": "일정과 운영 편의성을 같이 보고 선택했습니다.",
        "isSkipped": false
      },
      "runtimeFollowupQuestion": {
        "questionOrder": 2,
        "questionType": "follow_up",
        "questionText": "그 접근 방식을 선택한 이유를 조금 더 구체적으로 설명해주실 수 있나요?",
        "difficultyLevel": "medium"
      },
      "runtimeFollowupAnswer": {
        "answerText": "운영팀 적응 비용도 고려했습니다.",
        "isSkipped": false
      }
    }
  ]
}

출력:
{
  "followUpQuestions": [
    {
      "questionType": "follow_up",
      "difficultyLevel": "medium",
      "questionText": "그 판단 기준을 예시와 함께 설명해주실 수 있나요?",
      "parentQuestionOrder": 2
    }
  ],
  "qualityFlags": []
}

        "isSkipped": false
      }
    }
  ]

출력:
{
  "followUpQuestions": [],
