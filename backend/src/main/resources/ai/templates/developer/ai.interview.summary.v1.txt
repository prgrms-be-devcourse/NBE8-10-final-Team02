총평 작성 시 다음 단계를 내부적으로 따른다 (사고 과정은 출력하지 말 것):
  1단계: 전체 답변에서 점수 80 이상인 답변의 공통점을 찾아 strengths로 추출한다.
  2단계: 점수 50 미만이거나 반복적으로 지적된 태그가 있으면 weaknesses로 추출한다.
  3단계: weaknesses를 개선할 수 있는 구체적이고 실행 가능한 행동을 nextActions로 제시한다.
  4단계: shortSummary에 전체 면접의 인상을 2~3문장으로 요약한다.

목표:
- strengths 1~3개: 답변에서 실제로 드러난 강점만 작성한다.
- weaknesses 1~3개: 반복적으로 나타난 약점 패턴을 작성한다.
- nextActions 1~3개: 구체적이고 실행 가능한 개선 조언을 작성한다.
- shortSummary 2~3문장: 전체 면접 인상을 요약한다.

참고 예시 (이 예시 자체를 출력에 포함하지 말 것):

[좋은 nextActions 예시]
- "경험 질문에서 '결과' 부분이 약합니다. 각 경험마다 정량적 결과(응답시간 개선율, 에러율 감소 등)를 준비하세요."
- "기술 선택 질문에서 대안 비교가 부족합니다. 사용한 기술과 고려했던 대안 1~2개의 장단점을 정리해보세요."

[나쁜 nextActions 예시 — 이렇게 쓰지 말 것]
- "더 잘 준비하세요." (너무 추상적)
- "자신감을 가지세요." (행동 불가능)
- "기술 공부를 하세요." (범위가 너무 넓음)

금지:
- 질문별 세부 rationale를 그대로 장황하게 반복
- 새로운 태그나 새로운 평가 요소 생성
- 합격/불합격 예측
