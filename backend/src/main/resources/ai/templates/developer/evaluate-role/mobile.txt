[직무별 평가 기준: 모바일 개발자]

이 답변자의 직무는 모바일 개발자이다. 아래 기준을 평가에 추가 적용한다.

domainDepth 축 해석:
- 앱 생명주기 관리, 메모리/배터리 최적화, 네이티브 API 활용, 오프라인 처리, UI 반응성에 대한 깊이를 중점 평가한다.
- experience 질문: 모바일 성능 지표(ANR율, 콜드 스타트 시간, 메모리 사용량, 프레임 드롭)의 구체적 수치와 개선 과정을 기대한다.
- technical_stack 질문: 네이티브/크로스플랫폼 선택, 아키텍처 패턴(MVVM, MVI), 의존성 주입 프레임워크 등의 trade-off를 기대한다.
- technical_cs 질문: 스레드/코루틴, 메모리 관리, 네트워크 최적화 등 모바일 환경 특유의 제약에서의 적용을 기대한다.

직무 특화 참고 예시 (이 예시 자체를 출력에 포함하지 말 것):

[모바일 - 좋은 답변 평가 - experience]
질문(experience): "앱 성능을 최적화한 경험이 있나요?"
답변: "리스트 화면에서 스크롤 시 프레임 드롭이 발생했습니다. Systrace로 분석한 결과 아이템 바인딩 시 이미지 디코딩이 메인 스레드에서 실행되고 있었습니다. Glide의 비동기 로딩과 ViewHolder 재활용을 적용하고, DiffUtil로 변경된 아이템만 업데이트하여 평균 프레임률을 45fps에서 60fps로 개선했습니다."
→ score: 83
→ rationale: "①Systrace 기반 병목 분석으로 메인 스레드의 이미지 디코딩이라는 정확한 원인을 파악하고, DiffUtil 적용으로 프레임률을 33% 개선한 과정이 체계적이다. ②메모리 캐시 정책과 이미지 리사이징 전략, 그리고 저사양 기기에서의 성능 차이에 대한 고려가 부재하다. ③LRU 캐시 사이즈 설정 근거와 다양한 화면 밀도에서의 이미지 다운샘플링 전략을 보완하라."
