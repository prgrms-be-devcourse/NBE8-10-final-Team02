domainDepth 축 해석:
- 데이터 파이프라인 설계, 모델 학습/배포(MLOps), 피처 엔지니어링, 데이터 품질 관리에 대한 깊이를 중점 평가한다.

질문(technical_stack): "데이터 파이프라인을 어떻게 설계했나요?"
답변: "일 5억 건의 이벤트 로그를 처리하기 위해 Kafka로 수집하고, Spark Structured Streaming으로 실시간 집계한 뒤 Delta Lake에 적재했습니다. 스키마 진화를 지원하기 위해 Delta Lake의 schema evolution 기능을 활용했고, 데이터 품질 검증은 Great Expectations로 파이프라인 중간에 게이트를 두어 불량 데이터가 다운스트림에 전파되지 않도록 했습니다."
→ score: 84
→ rationale: "①일 5억 건 규모에서 Kafka-Spark-Delta Lake 조합을 선택하고, 스키마 진화와 데이터 품질 게이트를 설계한 판단이 체계적이다. ②Spark 클러스터의 리소스 산정 근거와 장애 시 데이터 유실 방지를 위한 체크포인팅 전략이 언급되지 않았다. ③Exactly-once 보장을 위한 Kafka 오프셋 관리 방식과 Delta Lake의 compaction/vacuum 운영 전략을 보완하라."
