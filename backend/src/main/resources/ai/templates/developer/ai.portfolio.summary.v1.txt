목표:
- repository와 문서에서 프로젝트/경험 단위 evidence를 추출한다.
- 비슷한 commit 메시지는 묶고, 의미 없는 commit은 제거하거나 가중치를 낮춘다.
- 문서와 GitHub에 동시에 존재하는 정보는 하나의 evidence로 통합할 수 있다.

출력 규칙:
- 반드시 아래 JSON 스키마를 정확히 따른다. 필드명 오타나 snake_case 변형 금지.
- signals는 최대 6개, evidenceBullets는 최대 5개로 제한한다.
- confidence는 "high", "medium", "low" 중 하나만 사용한다.
- 근거가 약하면 qualityFlags에 "low_context" 또는 "weak_evidence"를 넣는다.
- 설명 텍스트 없이 JSON만 출력한다.

출력 JSON 스키마 (필드명을 정확히 사용할 것):
{
  "projects": [
    {
      "projectKey": "string — 영문 소문자+하이픈, 예: linebot-backend",
      "projectName": "string — 사람이 읽기 좋은 프로젝트 이름",
      "summary": "string — 2~3문장 요약",
      "signals": ["string", "..."],
      "evidenceBullets": ["string", "..."],
      "confidence": "high | medium | low",
      "sourceRefs": ["파일경로 또는 커밋SHA", "..."],
      "qualityFlags": ["low_context | weak_evidence", "..."]
    }
  ],
  "globalStrengths": ["string", "..."],
  "globalRisks": ["string", "..."],
  "qualityFlags": ["string", "..."]
}
