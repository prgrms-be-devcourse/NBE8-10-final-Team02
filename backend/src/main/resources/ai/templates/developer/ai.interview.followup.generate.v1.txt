조건:
- currentAnswer.isSkipped가 true면 follow-up을 만들지 않는다.
- 답변 길이가 너무 짧거나 실질 정보가 없으면 null 결과를 반환할 수 있다.
- maxDepth를 넘기지 않는다.
- 반드시 JSON object 하나만 출력한다.
- follow-up을 생성하지 않는 경우에도 반드시 `followUpQuestion` 키를 유지하고 값은 null로 반환한다.
- 지정된 키 이름 외 다른 키를 추가하지 않는다.

좋은 follow-up 기준:
- 선택 이유의 trade-off를 묻는다.
- 구현 세부, 성능 영향, 대안 비교를 묻는다.
- 경험 질문이면 결과 근거와 의사결정 기준을 묻는다.

금지:
- 예/아니오형 단답 질문
- 기존 질문을 거의 그대로 반복
- 전혀 새로운 기술 주제로 점프
- 마크다운, 코드 블록, 설명 문장, 서론, 후기 출력

응답 JSON 구조 (반드시 이 키 이름과 구조를 정확히 따라라):
{
  "followUpQuestion": {
    "questionType": "follow_up",
    "difficultyLevel": "medium",
    "questionText": "그 선택을 했을 때 다른 대안과 비교한 기준을 조금 더 구체적으로 설명해주실 수 있나요?",
    "parentQuestionOrder": 3
  },
  "qualityFlags": []
}

생성하지 않는 경우:
{
  "followUpQuestion": null,
  "qualityFlags": ["low_context"]
}
