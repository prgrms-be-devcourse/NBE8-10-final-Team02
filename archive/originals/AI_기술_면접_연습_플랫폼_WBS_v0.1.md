# AI 기술 면접 연습 플랫폼 WBS

## 1. 문서 개요

- 버전: v0.1
- 기준 문서:
  - AI 기술 면접 연습 플랫폼 요구사항 명세서 v0.2
  - AI 기술 면접 연습 플랫폼 ERD 초안 v0.1
  - 역할 분담안
- 작성 목적: MVP 범위 기준으로 프로젝트의 작업 단위를 구조화하고, 구현 우선순위와 산출물을 정리하여 일정 관리 및 역할 분담의 기준으로 사용한다.

## 2. 작성 전제

- 팀 구성: 4인
- 개발 범위: 1차 MVP
- 개발 성격: 백엔드 중심 팀 프로젝트
- WBS 기준 기간: 4주 내외 진행을 가정한 초안
- 문서 상태 반영:
  - 프로젝트 아이디어 정리: 완료
  - 역할 분담안: 완료
  - 요구사항 명세서 초안: 완료
  - ERD 초안: 완료
  - DB 설계 정리 및 DDL 초안: 작성 완료, 필수 문서 아님

## 3. 범위 기준

### 3.1 MVP 포함 범위

- OAuth2 로그인
- GitHub 연동 및 repository 목록 조회
- 사용자 본인 commit 수집
- 이력서 및 기타 증빙 문서 업로드
- 문서 텍스트 추출 및 포트폴리오 데이터 저장
- 직무별 템플릿 기반 맞춤 자소서 생성
- 포트폴리오 기반 면접 질문 생성
- 텍스트 기반 모의 면접
- 면접 결과 저장 및 히스토리 조회

### 3.2 MVP 제외 범위

- PR 및 Issue 수집
- 음성 및 영상 기반 모의 면접
- 결제, 광고, 토큰 시스템
- 대규모 통계 및 커뮤니티 기능
- 게임화 보상 시스템

## 4. 마일스톤

- M2. 공통 기반 및 인증 완료

## 8. 리스크 메모

- OAuth2 제공자 3종 연동은 초기에 인증 흐름이 흔들릴 수 있으므로 공통 구조를 먼저 맞춘다.
- GitHub API 연동은 rate limit, 권한, pagination 이슈가 있으므로 PR/Issue는 MVP에서 제외한 현재 결정 유지가 중요하다.
- PDF 텍스트 추출은 품질 편차가 커서 실패 케이스와 사용자 메시지를 반드시 분리해 처리해야 한다.
- AI 생성 기능은 품질과 비용을 함께 봐야 하므로 프롬프트 입력 구조와 저장 구조를 너무 늦게 정하면 재작업이 커질 수 있다.
- 면접 세션은 단순 CRUD보다 상태 흐름이 중요하므로 세션 시작/진행/종료 기준을 먼저 확정한다.
