# AI 기술 면접 연습 플랫폼 화면 정의서

## 1. 문서 개요

- 문서명: AI 기술 면접 연습 플랫폼 화면 정의서
- 버전: v0.1
- 작성 목적: 현재 확정된 요구사항, ERD, DB 설계를 기준으로 MVP 화면 구조와 화면별 목적, 구성요소, 상태, 전환 흐름을 정의한다.
- 적용 범위: 1차 MVP 화면 정의
- 기준 문서:
  - AI 기술 면접 연습 플랫폼 요구사항 명세서 v0.2
  - AI 기술 면접 연습 플랫폼 ERD 초안 v0.1
  - AI 기술 면접 연습 플랫폼 DB 설계 정리 v0.1

## 2. 화면 설계 기준

### 2.1 서비스 흐름 기준

본 서비스의 핵심 사용자 흐름은 다음 순서를 따른다.

1. 로그인
2. 포트폴리오 등록
3. 자소서 생성
4. 면접 질문 생성
5. 텍스트 모의 면접
6. 결과 확인 및 히스토리 조회

### 2.2 도메인 구조 반영 원칙

- 로그인 계정과 GitHub 연동 계정은 분리해 다룬다.
- 포트폴리오 데이터는 GitHub 저장소/커밋과 업로드 문서로 나누어 화면도 분리한다.
- 자소서 생성과 면접 질문 생성의 기준 축은 `application`이다.
- 질문 생성 이력과 실제 모의 면접 세션 이력은 별도 화면 단위로 구분한다.

### 2.3 사용자 노출 용어 기준

- 내부 도메인 `application`은 화면에서 `지원 준비`로 표기한다.
- `interview_question_set`은 화면에서 `질문 세트`로 표기한다.
- `documents`는 화면에서 `업로드 문서` 또는 `포트폴리오 문서`로 표기한다.

### 2.4 MVP 범위 반영

이번 화면 정의서는 아래 범위만 포함한다.

- OAuth2 로그인
- GitHub 연동 및 repository 목록 조회
- 사용자 본인 commit 수집
- 문서 업로드 및 텍스트 추출
- 맞춤 자소서 생성
- 면접 질문 생성
- 텍스트 기반 모의 면접
- 면접 결과 및 히스토리 조회

이번 화면 정의서에서 제외하는 범위는 아래와 같다.

- PR 및 Issue 수집 화면
- 음성 및 영상 면접 화면
- 결제, 광고, 토큰 관련 화면
- 커뮤니티, 통계, 게임화 보상 화면

## 4. 전역 UI 정책

### 4.1 로그인 전/후 레이아웃

로그인 전에는 단일 컬럼 중심의 진입 화면을 사용한다.

로그인 후에는 아래 전역 내비게이션을 기본으로 한다.

- 포트폴리오
- 지원 준비
- 면접 히스토리
- 내 정보

### 4.2 공통 상태 표현

모든 화면은 아래 상태를 공통으로 고려한다.

- 초기 로딩 상태
- 데이터 없음 상태
- 부분 성공 상태
- 전체 실패 상태
- 재시도 가능 상태

### 4.3 공통 액션 규칙

- 저장 액션은 버튼 라벨을 `저장`, `생성`, `시작`, `완료`처럼 명확한 동사로 표기한다.
- 삭제/해제/종료 같은 파괴적 액션은 확인 모달을 사용한다.
- 긴 작업은 진행 상태를 사용자에게 반드시 보여준다.
- 폼 검증 오류는 필드 하단에 인라인 메시지로 노출한다.

### 4.4 공통 문구 기준

- 생성 실패 시: `다시 시도해주세요` 문구와 재시도 버튼을 함께 제공한다.
- 데이터 부족 시: 단순 에러 대신 어떤 데이터가 부족한지 안내한다.
- 접근 제한 시: 왜 사용할 수 없는지와 다음 행동을 함께 보여준다.

## SCR-01 로그인 화면

### 목적

사용자가 GitHub, Google, Kakao 중 하나를 통해 로그인할 수 있게 한다.

### 진입 조건

- 비로그인 사용자
- 세션 만료 사용자

### 권장 라우트

- `/login`

### 주요 구성요소

- 서비스 한 줄 소개
- GitHub 로그인 버튼
- Google 로그인 버튼
- Kakao 로그인 버튼
- 개인정보/이용 안내 링크
- 로그인 오류 알림 영역

### 주요 사용자 행동

- 로그인 제공자 선택
- 로그인 재시도

### 화면 상태

- 기본 상태: 3개 로그인 버튼 활성화
- 처리 중 상태: 선택한 버튼 로딩 표시, 나머지 버튼 비활성화
- 실패 상태: 상단 인라인 에러 메시지 노출

### 검증/예외

- 지원하지 않는 제공자 선택은 불가
- OAuth 인증 취소 시 로그인 실패 메시지 노출
- 필수 사용자 정보 누락 시 재시도 안내

### 완료 후 이동

- 신규 사용자: SCR-02 최초 진입 온보딩 화면
- 기존 사용자: 포트폴리오 보유 여부에 따라 SCR-03 또는 SCR-08

### 비고

- 요구사항상 로그인 성공 후 메인 또는 온보딩으로 이동해야 하므로 첫 진입 분기 로직이 필요하다.

## SCR-02 최초 진입 온보딩 화면

### 목적

최초 로그인 사용자가 서비스 흐름과 필요한 선행 작업을 이해하고 포트폴리오 등록으로 진입하도록 유도한다.

### 진입 조건

- 신규 사용자
- 포트폴리오 데이터가 아직 없는 사용자

### 권장 라우트

- `/onboarding`

### 주요 구성요소

- 서비스 핵심 흐름 소개
- `GitHub 연결하기` CTA
- `GitHub URL로 시작하기` CTA
- `문서 업로드하기` CTA
- 현재 진행 단계 표시

### 주요 사용자 행동

- 포트폴리오 등록 시작
- 온보딩 건너뛰기

### 화면 상태

- 기본 상태: 핵심 단계 카드 노출
- 건너뛰기 상태: 제한 기능 안내 배너 노출

### 검증/예외

- 온보딩을 건너뛰어도 로그인은 유지한다.
- 포트폴리오가 없으면 자소서/면접 생성 CTA는 비활성 또는 제한 안내 상태로 보여준다.

### 완료 후 이동

- GitHub 연결하기: SCR-04
- 문서 업로드하기: SCR-06
- 건너뛰기: SCR-03

## SCR-03 포트폴리오 홈 화면

### 목적

현재 사용자의 포트폴리오 준비 상태를 한눈에 보여주고 다음 작업으로 이동시키는 허브 화면이다.

### 진입 조건

- 로그인 사용자

### 권장 라우트

- `/portfolio`

### 관련 데이터

- GitHub 연결 상태
- 선택된 repository 수
- 저장된 commit 수
- 업로드 문서 수
- 최근 동기화 시각

### 주요 구성요소

- GitHub 연결 상태 카드
- Repository 선택 현황 카드
- 문서 업로드 현황 카드
- 최근 동기화/추출 상태 요약
- `GitHub 연동/변경` 버튼
- `Repository 선택` 버튼
- `문서 업로드` 버튼
- `지원 준비 시작` 버튼

### 주요 사용자 행동

- GitHub 연결 또는 변경
- repository 선택/재선택
- 문서 업로드
- 지원 준비 시작

### 화면 상태

- 빈 상태: 아직 연결된 GitHub/문서 없음
- 부분 완료 상태: GitHub만 연결됨 또는 문서만 업로드됨
- 완료 상태: 최소 1개 이상의 usable source 보유
- 일부 실패 상태: commit 수집 실패, 텍스트 추출 실패 항목 존재

### 검증/예외

- usable source가 없으면 `지원 준비 시작` 버튼 비활성화
- 일부 실패 항목은 재시도 CTA 제공

### 완료 후 이동

- GitHub 관련 액션: SCR-04 또는 SCR-05
- 문서 관련 액션: SCR-06
- 지원 준비 시작: SCR-08 또는 SCR-09

## SCR-04 GitHub 연동 및 URL 입력 화면

### 목적

GitHub 계정을 OAuth로 연동하거나 GitHub URL 입력을 통해 대상 계정/저장소를 식별한다.

### 진입 조건

- 로그인 사용자

### 권장 라우트

- `/portfolio/github`

### 관련 데이터

- GitHub 연결 상태
- 연결된 GitHub 로그인 ID
- 최근 sync 상태

### 주요 구성요소

- 탭 1: `GitHub 계정 연동`
- 탭 2: `GitHub URL 입력`
- 연결 상태 요약 카드
- access scope 안내
- URL 입력 필드
- URL 검증 결과 프리뷰 영역
- `연결하기` 버튼
- `다음으로` 버튼

### 주요 사용자 행동

- OAuth 연동 시작
- GitHub URL 입력 및 검증
- 기존 GitHub 연결 변경 또는 해제

### 화면 상태

- 미연결 상태
- 연결 성공 상태
- URL 검증 성공 상태
- URL 검증 실패 상태
- sync pending/success/failed 상태

### 검증/예외

- GitHub URL 형식 검증 필요
- 접근 권한 없는 repository 또는 비공개 저장소는 안내 필요
- GitHub 로그인과 GitHub 연동이 분리되어 있으므로, 로그인 제공자가 GitHub가 아니어도 이 화면은 사용 가능해야 한다.

### 완료 후 이동

- 연결/검증 성공 후 SCR-05 이동

### 비고

- 화면에서 `로그인 계정`과 `포트폴리오용 GitHub 연결`을 혼동하지 않도록 설명 문구가 필요하다.

## SCR-05 Repository 선택 및 commit 수집 화면

### 목적

사용자가 자소서/면접 생성에 사용할 repository를 선택하고 사용자 본인 commit을 수집하도록 한다.

### 진입 조건

- GitHub 계정 연동 또는 URL 검증 완료

### 권장 라우트

- `/portfolio/repositories`

### 관련 데이터

- repository 목록
- repository visibility
- 기본 브랜치
- 선택 여부
- commit 수집 상태
- 사용자 본인 commit 수

### 주요 구성요소

- repository 목록 테이블 또는 카드 리스트
- 검색/정렬/필터
- 선택 체크박스
- repository별 상태 배지
- `선택 저장` 버튼
- `commit 수집 시작` 버튼
- `다음으로` 버튼

### 주요 사용자 행동

- repository 검색
- repository 선택/해제
- 선택 저장
- commit 수집 시작

### 화면 상태

- 목록 로딩 상태
- 목록 비어 있음 상태
- repository 선택 전 상태
- commit 수집 진행 중 상태
- commit 수집 완료 상태
- 일부 repository 실패 상태

### 검증/예외

- repository가 0개면 다음 단계로 넘어갈 수 없도록 막지 않고, 문서 업로드 기반 진행 경로를 함께 안내한다.
- 본인 commit 식별 실패 시 repository 단위 경고를 표시한다.
- private repository는 MVP 정책 범위를 넘을 수 있으므로 제한 메시지 또는 숨김 처리가 필요하다.

### 완료 후 이동

- 문서 업로드로 계속하기: SCR-06
- 포트폴리오 결과 확인: SCR-07

## SCR-06 문서 업로드 및 추출 상태 화면

### 목적

이력서, 수상기록, 기타 증빙 문서를 업로드하고 텍스트 추출 상태를 확인한다.

### 진입 조건

- 로그인 사용자

### 권장 라우트

- `/portfolio/documents`

### 관련 데이터

- document_type
- 원본 파일명
- MIME type
- 파일 크기
- extract_status
- uploaded_at
- extracted_at

### 주요 구성요소

- 파일 드래그 앤 드롭 영역
- 파일 선택 버튼
- 문서 유형 선택 드롭다운
- 업로드 문서 리스트
- 추출 상태 배지
- 추출 결과 미리보기 영역
- 실패 항목 재시도 버튼
- `포트폴리오 저장 결과 보기` 버튼

### 주요 사용자 행동

- 파일 업로드
- 문서 유형 지정
- 추출 결과 확인
- 실패 항목 재시도
- 문서 삭제

### 화면 상태

- 업로드 전 빈 상태
- 업로드 진행 중 상태
- 추출 pending 상태
- 추출 success 상태
- 추출 failed 상태

### 검증/예외

- 지원하지 않는 파일 형식 업로드 불가
- 허용 용량 초과 파일 업로드 불가
- 추출 실패 항목은 저장은 되었으나 usable source가 아님을 구분해 보여준다.

### 완료 후 이동

- 결과 확인: SCR-07
- 지원 준비 시작: SCR-08 또는 SCR-09

## SCR-07 포트폴리오 등록 결과 화면

### 목적

이번 등록 작업의 성공/실패 항목과 현재 사용 가능한 포트폴리오 소스를 요약해서 보여준다.

### 진입 조건

- GitHub 수집 또는 문서 업로드 작업 완료 후

## SCR-08 지원 준비 목록 화면

### 진입 조건

### 관련 데이터

- company_name
- job_role
- status

- 연결된 source 수

### 주요 구성요소

- 지원 준비 카드 또는 테이블
- 상태 필터
- 정렬 드롭다운

## SCR-09 지원 준비 생성/기본 정보 화면

### 목적

하나의 `지원 준비`를 생성하기 위한 기본 정보와 사용할 포트폴리오 소스를 선택한다.

### 진입 조건

- 포트폴리오 usable source 1개 이상 존재 권장

### 권장 라우트

- `/applications/new`
- `/applications/{applicationId}/edit`

### 관련 데이터

- application_title
- company_name
- job_role
- 선택된 repository 목록
- 선택된 document 목록

### 주요 구성요소

- 지원 준비 제목 입력 필드
- 회사명 입력 필드
- 직무 입력 또는 선택 필드
- 사용할 repository 선택 영역
- 사용할 document 선택 영역
- 선택 source 요약 카드
- `저장 후 자소서 작성` 버튼

### 주요 사용자 행동

- 기본 정보 입력
- 포트폴리오 source 선택
- 임시 저장

### 화면 상태

- 신규 작성 상태
- 수정 상태
- source 없음 경고 상태

### 검증/예외

- 직무는 필수 입력
- 회사명은 선택 입력
- source가 전혀 없으면 저장은 허용할 수 있으나 자소서 생성 버튼은 제한하는 방식을 권장

### 완료 후 이동

- 자소서 작성: SCR-10
- 목록으로 이동: SCR-08

### 비고

- 내부적으로는 application이 중심이며, 이 화면이 이후 자소서와 면접 질문 생성의 기준점이 된다.

- 자소서 문항 목록
- 생성 답변
- 사용자 수정 답변
- 생성 옵션

### 주요 구성요소

- 지원 준비 요약 헤더
- 직무별 기본 템플릿 섹션
- 자소서 문항 리스트
- 문항 추가/삭제 버튼
- 문체/분량/강조 포인트 옵션
- `초안 생성` 버튼
- 문항별 답변 편집 영역
- `저장` 버튼
- `면접 질문 만들기` 버튼

### 주요 사용자 행동

- 문항 입력
- 생성 옵션 선택
- 전체 초안 생성
- 생성 결과 수정
- 저장

### 화면 상태

- 작성 전 빈 상태
- 생성 전 준비 상태
- 생성 중 상태
- 생성 완료 상태
- 문항별 실패 상태

### 검증/예외

- 문항이 비어 있으면 생성 불가

## SCR-11 면접 질문 생성 화면

### 목적

저장된 자소서와 포트폴리오 데이터를 기반으로 질문 세트를 생성한다.

### 진입 조건

- 지원 준비 존재
- 포트폴리오 데이터 존재
- 자소서 초안 존재 또는 최소 입력 충족

### 권장 라우트

- `/interview/question-sets/new?applicationId={applicationId}`

### 관련 데이터

- application 정보
- 자소서 문항/답변 요약
- 질문 개수
- 난이도
- 질문 유형

### 주요 구성요소

- 지원 준비 요약 카드
- 질문 개수 선택
- 난이도 선택
- 질문 유형 멀티 선택
- `질문 생성` 버튼
- 생성 결과 미리보기 리스트
- `질문 세트 저장` 버튼
- `모의 면접 시작` 버튼

### 주요 사용자 행동

- 생성 옵션 선택
- 질문 세트 생성
- 질문 확인 후 저장
- 바로 모의 면접 시작

### 화면 상태

- 옵션 입력 전 상태
- 생성 중 상태
- 생성 완료 상태
- 품질 저하 안내 상태
- 생성 실패 상태

### 검증/예외

- 직무 미입력 시 생성 불가
- 질문 개수/난이도는 허용 범위 내에서만 선택 가능
- 데이터 부족 시 생성은 허용하되 품질 저하 안내 필요

### 완료 후 이동

- 질문 세트 상세: SCR-12
- 면접 시작: SCR-13

- 질문 세트 헤더

## SCR-13 텍스트 모의 면접 세션 화면

### 목적

질문을 순차 제시하고 사용자의 텍스트 답변을 저장하며 세션을 종료할 수 있게 한다.

### 진입 조건

- 질문 세트 존재
- 로그인 상태

### 권장 라우트

- `/interview/sessions/{sessionId}`

### 관련 데이터

- session status
- 현재 질문
- 전체 질문 수
- 현재 순번
- 답변 텍스트
- 건너뛰기 여부

### 주요 구성요소

- 진행률 표시바
- 현재 질문 카드
- 질문 유형/난이도 배지
- 답변 입력 textarea
- `답변 제출` 버튼
- `건너뛰기` 버튼
- `이전 답변 보기` 선택 기능
- `면접 종료` 버튼
- 세션 종료 확인 모달

### 주요 사용자 행동

- 질문 읽기
- 답변 입력 및 제출
- 건너뛰기
- 세션 종료

### 화면 상태

- 시작 전 준비 상태
- 진행 중 상태
- 답변 제출 중 상태
- 꼬리 질문 생성 중 상태
- 종료 확인 상태
- 종료 완료 상태

### 검증/예외

- 빈 답변 제출 불가
- 건너뛰기와 빈 제출은 구분 처리
- 세션 종료 전까지 결과 화면으로 이동 불가
- 질문/답변 저장 실패 시 현재 답변 보존 상태에서 재시도 제공

### 완료 후 이동

- 세션 완료 후 SCR-14
- 중도 종료 후에도 결과 생성 정책에 따라 SCR-14 또는 SCR-15

### 비고

- 꼬리 질문은 필요 시 생성 가능하므로, 세션 중 동적으로 질문 카드가 추가될 수 있는 구조를 고려한다.

- SCR-13 텍스트 모의 면접 세션

- 기존 지원 준비 열기
- SCR-10 또는 SCR-11 또는 SCR-15로 진입

## 7. 화면 설계 시 우선 결정이 필요한 항목

- 대시보드 첫 진입 화면을 `포트폴리오 홈`으로 할지 `지원 준비 목록`으로 할지
- 온보딩 건너뛰기를 허용할지 여부
- GitHub URL 입력 사용 시 repository 선택 범위를 어디까지 허용할지
- 자소서 문항 입력 화면에서 문항 추가/삭제 자유도를 얼마나 줄지
- 면접 세션 중 꼬리 질문을 실시간으로 추가할지, 생성된 질문 세트만 고정 사용할지
- 중도 종료 세션도 결과 화면을 만들지 여부
