# AI 기술 면접 연습 플랫폼 요구사항 명세서

이 문서는 사용자가 제안한 최신 기능 변경안을 기준으로 요구사항을 다시 정리한 구현 기준 문서다.
모호한 표현은 구현 가능하도록 정제했고, 여전히 판단이 갈릴 수 있는 항목은 `docs/project/open-items.md`에 검토 메모로 분리했다.

## 1. 주요 기능

### 1.1 기능 구성 개요

본 서비스의 핵심 기능은 아래 5개 영역으로 구성된다.

1. 포트폴리오 수집 및 저장
2. 맞춤 자소서 생성
3. 면접 질문 생성
4. 실시간 모의 면접
5. 회원가입 및 인증

확장 기능으로 유료 플랜, 통계, 동기부여 기능 등을 추후 고려한다.

### 1.2 필수 기능 정의

#### 1.2.1 포트폴리오 수집 및 저장 (FR-02)

##### 기능 설명

사용자가 GitHub URL 또는 GitHub OAuth 연동 정보와 이력서, 수상기록 등 문서를 입력하면 이를 수집 및 저장하여 이후 자소서 생성과 면접 기능의 기반 데이터로 활용한다.

##### 세부 기능

- GitHub URL 입력 시 public repository 목록 조회
- GitHub OAuth 연동 시 사용자 동의 범위 안에서 public/private repository 조회
- 사용자 본인 commit 내역 수집
- PR, Issue 수집은 후속 확장으로 분리하고 MVP 기본값은 commit 중심으로 운영
- 이력서 파일 업로드
- 수상기록 및 기타 증빙 파일 업로드
- PDF, DOCX, MD 텍스트 추출
- 추출 데이터 정규화 및 DB 저장
- 현재 계산 가능한 GitHub/문서 준비 현황 요약 조회
- 로그인 이후 공통 화면에서 준비 현황 축약 위젯 제공
- 포트폴리오 항목 수정 및 삭제

##### 입력

- GitHub URL 또는 GitHub OAuth 인증 정보
- 파일 - 이력서, 수상기록, 기타 증빙 문서

##### 출력

- 사용자 포트폴리오 프로필
- 저장된 repository, commit, 문서 텍스트 데이터
- 준비 현황 위젯/대시보드용 요약 데이터

##### 제약 조건

| 항목 | 조건 |
| --- | --- |
| 파일 형식 | PDF, DOCX, MD |
| 파일 크기 | 파일당 최대 10MB |
| 파일 개수 | 최대 5개 |
| repository 우선 노출 기준 | 최근 2년 내 활동 repository를 우선 정렬하고, 필요 시 전체 목록 조회 가능 |
| private repository 접근 | GitHub OAuth 추가 동의가 있는 경우에만 허용 |
| 준비 현황 요약 미구현 값 | 가짜 숫자 대신 `null`과 준비 중 상태로 표시 |

##### 예외 처리

- 이미지 기반 스캔 PDF는 기본 파서로 텍스트 추출이 실패할 수 있다.
- OCR 기반 추출은 MVP 기본 범위에서 제외하고, 추후 별도 확장으로 검토한다.
- 지원하지 않는 파일 형식 업로드
- GitHub URL 형식 오류
- 파일 크기 초과
- GitHub API 호출 실패
- 동일 문서 재업로드 시 사용자 확인 후 덮어쓰기 또는 업로드 취소

##### 기대 효과

- 반복 입력 없이 이후 기능에서 재사용 가능
- 사용자 경험 기반의 일관된 AI 결과 생성 가능

#### 1.2.2 맞춤 자소서 생성 (FR-03)

##### 기능 설명

사용자의 포트폴리오 정보와 지원 회사, 직무, 지원 유형 정보를 기반으로 맞춤형 자소서 초안을 생성한다.

##### 세부 기능

- 회사명, 직무명, 지원 유형 입력
- 자소서 문항 직접 입력 또는 템플릿 선택
- 직무별 기본 템플릿 적용
- 회사별 특수 문항은 사용자가 직접 추가 입력
- 포트폴리오 기반 강점 추출 및 자소서 초안 생성
- 생성 결과 저장 및 재조회
- 동일 포트폴리오로 복수 지원 버전 관리

##### 자소서 문항 템플릿 관리 방식

- 직무별 자주 묻는 문항을 서비스 내 기본 템플릿으로 제공한다.
- 사용자는 문항을 직접 추가하거나 수정할 수 있다.
- 회사별 특수 문항은 사용자가 직접 입력한다.

##### 입력

- 지원 회사명
- 지원 직무명
- 지원 유형
- 자소서 문항 - 직접 입력 또는 템플릿 선택
- 사용자 포트폴리오 데이터 - 기저장 데이터 자동 참조

##### 출력

- 문항별 자소서 초안
- 생성 일시와 이력을 포함한 저장 결과

##### 제약 조건

| 항목 | 조건 |
| --- | --- |
| 문항 수 | 1회 생성 시 최대 10문항 |
| 재생성 | 동일 문항 재생성 가능, 이전 결과는 이력 보존 |
| 저장 보존 기간 | 회원 탈퇴 처리 전까지 보존, 탈퇴 후 30일 유예 뒤 완전 삭제 |
| 기본 분량 옵션 | short, medium, long |

##### 예외 처리

- 포트폴리오 미등록 상태에서 생성 요청
- AI API 호출 실패
- AI API 응답 지연 - 10초 이상 시 타임아웃 또는 재시도 안내
- 동시 생성 요청 중복

##### 예시 활용

- 오픈소스 지원용 자소서
- 대기업 및 IT 서비스 기업 맞춤 자소서
- 동일 사용자의 여러 직무 지원 버전 생성

#### 1.2.3 면접 질문 생성 (FR-04)

##### 기능 설명

생성된 자소서와 포트폴리오 데이터를 바탕으로 예상 면접 질문과 꼬리 질문을 생성한다.

##### 세부 기능

- 자소서 기반 예상 질문 생성
- 프로젝트 경험 기반 기술 질문 생성
- CS 관련 기초 질문 생성
- 꼬리 질문 생성
- 질문 카테고리 분류
- 생성된 질문 개별 삭제 및 수동 추가

##### 질문 유형 예시

- 자기소개 및 지원동기
- 프로젝트 경험 질문
- 기술 선택 이유
- 문제 해결 경험
- CS 기초
- 꼬리 질문

##### 입력

- 자소서 내용 - 기저장 데이터 자동 참조
- 포트폴리오 데이터 - 기저장 데이터 자동 참조
- 지원 직무 정보

##### 출력

- 예상 면접 질문 리스트
- 질문 유형 및 난이도 정보

##### 제약 조건

| 항목 | 조건 |
| --- | --- |
| 생성 질문 수 | 1회 최대 20개 |
| 재생성 | 전체 또는 개별 질문 단위 재생성 가능 |

##### 예외 처리

- 자소서 미생성 상태에서 요청
- AI API 호출 실패
- 입력 데이터 부족으로 질문 품질이 떨어질 수 있는 경우 품질 저하 안내

##### 기대 효과

- 자소서 작성과 면접 준비가 자연스럽게 연결됨
- 사용자 경험 기반의 개인화된 질문 제공 가능

#### 1.2.4 실시간 모의 면접 (FR-05, FR-06)

##### 기능 설명

사용자가 1차 범위에서 실시간 음성 입력 방식으로 모의 면접을 진행하면, 서비스가 질문을 제시하고 사용자가 최종 확인한 답변 텍스트를 기록하며 종료 후 피드백 리포트를 제공한다. 음성 기능이 불안정하거나 사용할 수 없는 경우에는 같은 세션에서 텍스트 입력 fallback을 허용한다.

##### 세부 기능

- 면접 세션 생성
- 질문 순서 제어
- 마이크 권한 요청 및 장치 상태 확인
- 실시간 음성 입력 수집
- 실시간 transcript 표시
- transcript 확인/수정 후 답변 제출
- 음성 장애 시 같은 세션에서 텍스트 입력 fallback
- 세션 상태 관리
- 종료 후 피드백 리포트 생성 및 제공
- 세션 히스토리 조회

##### 세션 상태

- 시작 전
- 진행 중
- 일시중지
- 종료
- 피드백 완료

##### 피드백 리포트 항목

- 답변 완성도
- 기술 정확도
- 구체성
- 개선 제안
- 종합 코멘트

##### 입력

- 면접 주제 또는 직무
- 질문 리스트
- 실시간 음성 입력 또는 텍스트 fallback 답변

##### 출력

- 세션별 질문, 최종 확인 답변 기록
- 피드백 리포트
- 히스토리 데이터

##### 제약 조건

| 항목 | 조건 |
| --- | --- |
| 답변 최소 글자수 | 50자 이상, 단 건너뛰기 응답은 예외 |
| 답변 최대 글자수 | 1,000자 이하 |
| 동시 세션 | 사용자 1인당 동시 진행 세션 1개 제한 |
| 세션 질문 수 | 최소 3개, 최대 20개 |
| 세션 복원 | 브라우저 이탈 또는 새로고침 후 30분 이내 복원 |
| 입력 모드 | 1차 범위는 실시간 음성 입력을 기본으로 하고, 음성 기능 실패 시 같은 세션의 텍스트 입력 fallback을 허용 |
| 질문 출력 방식 | v1은 텍스트 질문 카드만 제공하고 TTS는 포함하지 않음 |
| 원본 음성 저장 | v1은 raw audio를 저장하지 않음 |

##### 예외 처리

- 세션 진행 중 브라우저 이탈 또는 새로고침 시 마지막 저장 상태로 복원
- 30분 이상 답변 미입력 시 세션 자동 일시정지
- 동일 사용자 중복 세션 생성 시 기존 세션으로 이동 유도
- 마이크 권한 거부 또는 장치 없음 시 텍스트 입력 fallback 제공
- 실시간 STT 실패 또는 지연 시 세션은 유지하고 텍스트 입력 fallback 또는 재시도 안내 제공
- 피드백 생성 AI 호출 실패 시 세션 기록은 보존하고 다시 시도 안내
- 답변 글자수 미달 시 제출 차단

##### 향후 확장

- 영상 면접
- 태도, 말투 분석

#### 1.2.5 회원가입 및 인증 (FR-01)

##### 기능 설명

사용자는 OAuth2 기반 소셜 로그인을 통해 회원가입 및 로그인할 수 있으며, 이후 자신의 포트폴리오와 생성 이력을 관리한다.

##### 세부 기능

- GitHub 로그인
- Google 로그인
- Kakao 로그인
- 사용자 프로필 관리
- 권한 및 인증 토큰 관리
- 탈퇴 유예 및 데이터 삭제 일정 관리

##### 입력

- OAuth2 인증 정보

##### 출력

- 사용자 계정
- 인증 토큰
- 사용자 기본 프로필

##### 목적

- 외부 연동을 간편하게 만들고 사용자 진입장벽을 낮춤
- GitHub 연동 기능과 자연스럽게 연결 가능

##### 예외 처리

| 상황 | 처리 방식 |
| --- | --- |
| OAuth 인증 실패 | 에러 안내 후 로그인 페이지 유지 |
| 토큰 만료 | Refresh Token으로 자동 재발급, 실패 시 재로그인 유도 |
| 이미 가입된 이메일로 다른 소셜 로그인 시도 | 자동 병합하지 않고 기존 계정 안내 또는 계정 연결 절차로 유도 |
| 회원 탈퇴 | 30일 유예 후 데이터 완전 삭제 |

## 2. 서비스 범위

| 구분 | 포함 | 미포함 |
| --- | --- | --- |
| 사용자 | 개발자 취준생, IT 직군 이직 및 취직 준비자 | 비개발 직군 |
| 인증 | OAuth2 소셜 로그인 | 이메일, 비밀번호 자체 회원가입 |
| 포트폴리오 | GitHub, PDF, MD, DOCX 파일 | Notion, 블로그 등 기타 URL |
| 자소서 | 텍스트 기반 초안 생성 | 최종 제출용 완성본 자동 제출 |
| 면접 | 실시간 음성 입력 기반 모의 면접, 텍스트 fallback | 영상 면접, 태도·말투 분석 |

## 3. 사용자 흐름

회원가입 또는 로그인
→ 포트폴리오 등록 - GitHub URL 또는 OAuth 연동 + 파일 업로드
→ 자소서 생성 - 회사, 직무, 지원 유형 입력 + 문항 선택
→ 면접 질문 생성 - 자소서 기반 자동 생성
→ 모의 면접 진행 - 실시간 음성 입력 또는 텍스트 fallback
→ 피드백 리포트 확인 및 히스토리 조회

## 4. 이번 반영에서 함께 정리한 기준

- GitHub URL 입력만으로는 public repository만 접근 가능하다.
- private repository 접근은 GitHub OAuth 추가 동의가 있을 때만 허용한다.
- PR, Issue 수집은 이번 버전 canonical 기준에서 MVP 제외 범위로 둔다.
- 최근 2년 repository 규칙은 하드 필터가 아니라 우선 정렬 기준으로 적용한다.
- 동일 이메일의 다른 소셜 로그인은 자동 병합하지 않는다.
- 스캔 PDF OCR 처리는 향후 확장으로 둔다.
