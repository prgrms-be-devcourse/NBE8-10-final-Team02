# AI 기술 면접 연습 플랫폼 제품 정의서

이 문서는 서비스의 방향, 핵심 가치, MVP 범위, 확장 기준을 제품 관점에서 잠그는 상위 문서다.
세부 기능 계약은 `docs/requirements.md`, `openapi.yaml`, `docs/db/schema.md`를 따른다.

## 1. 제품 한 줄 정의

개발자 취업 준비생이 자신의 GitHub 활동과 문서형 포트폴리오를 한 번 등록하면, 이를 바탕으로 자소서 작성부터 면접 질문 생성, 텍스트 기반 모의 면접, 피드백 히스토리 관리까지 한 흐름으로 이어서 사용할 수 있게 하는 AI 기반 취업 준비 서비스.

## 2. 제품이 해결하려는 문제

### 2.1 현재 사용자 불편
- GitHub 활동, 이력서, 수상기록, 지원 회사 정보를 매번 다시 입력해야 한다.
- 자소서 작성과 면접 준비가 서로 다른 도구로 분리되어 준비 흐름이 끊긴다.
- 일반 채팅형 AI는 프로젝트 근거와 문서 맥락을 안정적으로 재사용하기 어렵다.
- 개발자에게 중요한 repository 활동, commit 경험, 기술 선택 이유가 구조화되어 남지 않는다.
- 모의 면접 결과가 누적되지 않아 반복 학습과 약점 추적이 어렵다.

### 2.2 우리가 제공하는 해결 방식
- 포트폴리오를 한 번 수집하고 구조화하여 이후 모든 기능에서 재사용한다.
- 자소서 생성, 예상 질문 생성, 모의 면접, 피드백 확인을 하나의 데이터 흐름으로 연결한다.
- 개발자 취업 준비에 중요한 GitHub 활동과 문서형 포트폴리오를 함께 다룬다.
- 면접 결과를 점수, 태그, 코멘트 형태로 저장해 다음 연습에 반영할 수 있게 한다.

## 3. 타겟 사용자

- 부트캠프 수료 후 취업 준비 중인 개발자
- 신입 개발자 취업 준비생
- 이직을 준비 중인 주니어 개발자
- 자신의 GitHub, 프로젝트 경험, 이력서를 근거로 맞춤형 자소서와 면접 준비를 하고 싶은 사용자

비대상은 아래와 같다.
- 비개발 직군 취업 준비자
- 이메일/비밀번호 자체 회원가입이 필요한 사용자
- 음성/영상 면접이 필수인 사용자

## 4. 제품 핵심 가치

### 4.1 포트폴리오를 재사용 가능한 데이터 자산으로 만든다
단발성 입력이 아니라, 저장 가능한 사용자 자산으로 다룬다.

### 4.2 자소서와 면접 준비를 하나의 여정으로 연결한다
자소서가 면접 질문의 입력이 되고, 면접 결과가 다시 개선 포인트가 된다.

### 4.3 개발자 도메인에 맞는 근거 중심 생성 경험을 제공한다
일반적인 이력 정보뿐 아니라 GitHub 활동과 프로젝트 문맥을 함께 활용한다.

### 4.4 결과 기록 기반 반복 학습을 가능하게 한다
세션별 기록과 태그를 남겨 사용자가 약점을 추적할 수 있게 한다.

## 5. 핵심 사용자 흐름

1. 사용자는 GitHub, Google, Kakao 중 하나로 로그인한다.
2. 사용자는 GitHub URL을 입력하거나 GitHub OAuth를 연동하고, 이력서/증빙 문서를 업로드한다.
3. 시스템은 repository 정보, 사용자 본인 commit, 문서 텍스트를 수집하여 포트폴리오 데이터로 저장한다.
4. 사용자는 지원 회사, 직무, 지원 유형, 문항을 입력하고 자소서 초안을 생성한다.
5. 시스템은 저장된 포트폴리오와 자소서를 바탕으로 예상 면접 질문을 생성한다.
6. 사용자는 질문 세트를 선택해 텍스트 기반 모의 면접을 진행한다.
7. 시스템은 세션 종료 후 점수, 태그, 개선 코멘트를 포함한 피드백을 제공하고 히스토리에 저장한다.

## 6. MVP에서 잠그는 제품 결정

### 6.1 인증과 계정
- OAuth2 로그인은 GitHub, Google, Kakao만 지원한다.
- 이메일/비밀번호 자체 회원가입은 MVP 범위에서 제외한다.
- 다른 소셜 로그인으로 같은 이메일이 들어와도 자동 병합하지 않고 안내 또는 연결 절차로 처리한다.

### 6.2 포트폴리오 수집
- GitHub URL 입력 시 public repository 기준으로 조회한다.
- private repository는 GitHub OAuth 연동과 추가 동의가 있을 때만 허용한다.
- MVP에서 기본 수집 대상은 repository와 사용자 본인 commit이다.
- PR, Issue 수집은 확장 기능으로 남긴다.
- 지원 문서 형식은 PDF, DOCX, MD이다.
- 파일 크기는 파일당 최대 10MB, 사용자당 기본 최대 5개다.
- 최근 2년 활동 repository를 우선 정렬하되, 하드 제외 기준으로 쓰지 않는다.

### 6.3 자소서 생성
- 직무별 기본 템플릿을 우선 제공한다.
- 회사별 특수 문항은 사용자가 직접 입력한다.
- 한 번에 생성 가능한 문항 수는 최대 10개다.
- 동일 포트폴리오로 복수의 지원 버전을 저장하고 재조회할 수 있다.

### 6.4 면접 질문 생성
- 자소서와 포트폴리오를 기반으로 예상 질문, 기술 질문, 꼬리 질문을 생성한다.
- 한 번에 생성 가능한 질문 수는 최대 20개다.
- 질문 유형과 난이도는 저장 가능한 메타데이터로 관리한다.

### 6.5 텍스트 기반 모의 면접
- 세션은 텍스트 채팅 방식으로 진행한다.
- 사용자당 동시에 진행 가능한 활성 세션은 1개다.
- 답변 길이는 일반 답변 기준 50자 이상 1000자 이하로 제한한다.
- 30분 이상 답변이 없으면 세션을 자동 일시정지하고 재개할 수 있게 한다.
- 세션 결과는 질문/답변 기록, 점수, 태그, 코멘트와 함께 저장한다.

## 7. MVP 범위

### 7.1 포함 범위
- OAuth2 로그인
- GitHub repository 목록 조회
- 사용자 본인 commit 수집
- 문서 업로드와 텍스트 추출
- 포트폴리오 데이터 저장
- 직무별 템플릿 기반 자소서 생성
- 포트폴리오 기반 면접 질문 생성
- 텍스트 기반 모의 면접
- 면접 결과 저장 및 히스토리 조회

### 7.2 제외 범위
- PR, Issue 수집
- 음성/영상 기반 모의 면접
- 결제, 광고, 토큰 시스템
- 커뮤니티, 게시판, 합격 후기
- 대규모 통계, 추천, 동기부여 시스템
- Notion, 블로그 등 기타 URL 수집

## 8. 성공 기준

### 8.1 제품 성공 지표 초안
- 회원가입 후 첫 포트폴리오 등록 완료율
- 포트폴리오 등록 사용자 중 자소서 첫 생성 전환율
- 자소서 생성 사용자 중 면접 질문 생성 전환율
- 면접 질문 생성 사용자 중 모의 면접 완료율
- 생성 실패율과 세션 중도 이탈률

### 8.2 품질 성공 기준
- 업로드/동기화/생성 실패가 부분 실패로 식별되고 재시도 가능해야 한다.
- 같은 사용자 흐름에서 이미 저장된 데이터가 손실되지 않아야 한다.
- 기능 제한, 상태값, 개수 제약이 화면, API, DB 문서에 일관되게 드러나야 한다.

## 9. 차별점

### 9.1 단순 채팅형 AI와의 차이
단발성 프롬프트가 아니라 포트폴리오를 저장하고 재사용한다.

### 9.2 개발자 특화 흐름
GitHub 활동과 문서형 포트폴리오를 동시에 다루므로 일반 자소서 생성 서비스보다 개발자 도메인 적합성이 높다.

### 9.3 기록 기반 반복 학습
면접 결과를 누적하여 개인별 개선 흐름을 만들 수 있다.

## 10. 주요 리스크와 대응

### 10.1 GitHub API 제약
- rate limit, scope 부족, pagination 이슈가 발생할 수 있다.
- MVP에서는 commit 중심으로 범위를 제한하고, 동기화 실패는 부분 실패로 관리한다.

### 10.2 문서 추출 품질
- 스캔 PDF나 비정형 문서는 추출 품질이 낮을 수 있다.
- 업로드 성공과 텍스트 추출 성공을 분리해 관리하고, 추출 실패 상태를 저장한다.

### 10.3 AI 비용과 품질 편차
- 생성 비용이 커지거나 결과 편차가 발생할 수 있다.
- 모델 추상화 계층, 템플릿 기반 생성, 재시도 정책으로 비용과 실패를 제어한다.

### 10.4 민감 데이터 보호
- 이력서, 증빙 문서, OAuth credential은 민감할 수 있다.
- 최소 수집, 접근 제어, 로그 마스킹, secret 분리를 기본 원칙으로 둔다.

## 11. 확장 방향

- GitHub PR, Issue 수집 확장
- 음성/영상 기반 모의 면접
- 약점 태그 기반 개인화 추천
- 면접 리포트 및 통계 대시보드
- 사용자별 사용량/비용 상한 기반 무료/유료 플랜
- 채용 공고 분석과 공고 맞춤 생성 고도화

## 12. 팀 프로젝트 관점의 가치

- OAuth2, GitHub API, 문서 처리, AI 파이프라인, 세션 상태 관리, 히스토리 저장 등 실무형 주제를 함께 다룰 수 있다.
- 기능 자체보다 문서-계약-구현을 함께 잠그는 AI 문서 주도 개발 실습 주제로도 적합하다.
- 백엔드 중심 팀이 공통 규칙, API 계약, 테스트, 관측성까지 함께 경험하기 좋다.
